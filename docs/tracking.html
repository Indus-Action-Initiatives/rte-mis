Student Tracking Management

In the Student tracking Module, schools can manage and monitor students' academic progress, attendance and dropout students tracking. The module provides a comprehensive view of student data, filtered by class and displayed in a grid format. This allows school administrators to efficiently track and update each student's information on the RTE-MIS.

This approach is useful for tracking students' performance, educational progress, and retention rates within the school. It also allows for evaluating the school's performance and achievement based on students' ranks or grades.

By monitoring dropout updates, we can identify the underlying reasons why students leave school. If a student transfers from one school to another, we investigate the reasons for their departure. This information helps us address the issues and work towards reducing the overall dropout rate in schools.

Our approach is also to use the student tracking module in two ways, one is internally for states who are approaching the whole RTE-MIS portal and the other is for States who are willing to use only the student tracking system.

In the case of external states, the system should import the list of schools and students into the student tracking system. It should then apply the same tracking update process used for internal states.

Reports need to be created for the tracking system to support retention surveys, monitor student dropout rates, and evaluate school performance.

There are four major activities to perform in this module:

  1. Import school information
  2. Existing Students Collection Panel
  3. Student Attendance and Performance Tracking
  4. Dropout Students Tracking

1. Import school information

  • This section will be followed by the States who are going to use only the student tracking system only from the RTE - MIS.
  • All information will be imported from the State. State admin will download a template to update school information, which they will then upload to the portal.
  • School information can be further modified as per requirement basis, from the portal itself. If they have to make changes on many school records then they can again upload the updated excel and update operation will perform for school information.
  • The required fields for schools are the same as those mentioned in point 3 new school registration section of the School Management.

2. Existing Students Collection Panel

  • In this panel, we will collect information for students not yet in RTE-MIS—specifically, those who were studying at the school before the portal was launched (offline students).
  • Information for the previous year’s students, organized by class, needs to be collected. (For example, if the portal starts in a state in the 2025-26 academic year, then information for students from the 2024-25 academic year needs to be collected.)
  • All information will be imported from the schools. Schools will download a template to update student information, which they will then upload to the portal. The same process will apply for schools in external states.
  • Schools can do further modification on the student information from the portal itself. If they have to make changes on many student records then they can again upload the updated excel and update operation will perform for students information.
  • The following fields are required for collecting student information:
    1. Udise Code: School’s udise code should be added.
    2. Current Year: The academic year for which the information is being collected should be specified.
    3. Entry Year: The year in which the student was admitted through this scheme (dropdown menu).
    4. Entry Class: The class into which the student was admitted (dropdown menu).
    5. Current Class: The class the student is currently in, based on the year specified in point a (dropdown menu).
    6. Medium: The medium of instruction the student is studying in (dropdown menu).
    7. Name of Student: The full name of the student.
    8. Date of Birth: The student’s date of birth, selected from a calendar.
    9. Parent Name: The name of one parent (father, mother, or guardian).
    10. Mobile Number: The mobile number of the parent whose name is entered above.
    11. Address: The full residential address of the student.
    12. Caste: The caste of the student.
    13. Gender: The gender of the student.
    14. Religion: The religion of the student. (should not be mandatory)

3. Student Attendance and Performance Tracking

  • After entering student data into the portal via the Student Collection Panel, schools will use the Student Attendance and Performance Tracking section to update attendance and performance records.
  • These updates should be made by the school on the basis of medium, class, and for all students who are studying at their school under this scheme.
  • To facilitate this, a panel needs to be created where schools can view the complete list of students and apply filters based on year, class and medium.
  • Schools will need to collect and update the attendance and performance data for each student on the portal.
  • The student list should be displayed in a grid format with columns for attendance, grades, ranking and total students (RTE + non RTE). These columns will include text fields where the school admin can enter details for individual students.
Notes:
  1. Schools are not required to specify the maximum number of school days; instead, this is set as an average of 222 days out of 365.
  2. Schools must provide the total number of students in each class and medium.
  3. The timeline for this process will follow the same schedule as the Reimbursement process. An option to set this timeline will be available on the Create Session Panel, allowing the state to define it for the year or enter a specific timeline if needed.
  4. By default maximum entry class should be class 8, but this should be configurable for state that if they want to show it till class 12.
  5. Students will be automatically promoted to the next class when a new admission cycle begins. This process will be applied globally, ensuring that all students are promoted to the next class for the upcoming academic year as soon as the new admission cycle starts.
  6. Year-on-year records should be maintained for reporting purposes.
  7. A separate table should be created to merge and store both old and new records, ensuring that historical data is preserved and easily accessible for future reporting.
  8. School can view the all year records but can edit one year back only.

4. Dropout Students Tracking

The Dropout Tracking Module is designed to efficiently manage and record student dropout cases, ensuring that schools can track and understand the reasons behind student dropouts. This module provides a streamlined process for marking students as dropouts with detailed information.

  • To mark a student as a dropout in the portal, follow these steps:
    1. Select the Academic Year: Choose the year in which the student has dropped out.
    2. Select the Class: Choose the class to narrow down the list of students.
  • Next, select the specific student you wish to mark as a dropout and click the "Dropout" button.
  • After clicking the "Dropout" button, a popup window will appear where you need to provide basic information about the student.
  • In the popup window, you must complete the following fields:
    Reason for Dropping Out: A dropdown menu will appear asking for the reason why the student is dropping out:
    1. Taking Admission to Another School:
      1. State Name: Enter the name of the state where the student is transitioning.
      2. District Name: Enter the name of the district in the transitioning state.
      3. School Name: Provide the name of the new school where the student will be admitted.
      4. Remarks: Include any additional information if necessary.
    2. Not Studying Anymore:
      1. Select Reason:Choose a reason from a dropdown menu that lists various causes for the student's lack of interest in continuing education. Reasons should include:
        1. Financial difficulties
        2. Family responsibilities
        3. Family migration
        4. Health issues
        5. Lack of interest in studies
        6. Transportation issues
        7. Bullying or harassment
        8. Academic struggles
        9. Student getting married
        10. Parental pressure
        11. Not satisfied with School
        12. Cultural or religious reasons
        13. Reason not Mentioned
      2. Remarks: Provide a detailed explanation of why the student is dropping out.

Finally, click the "Save" button. The selected student will then be marked as a dropout and will no longer be displayed in the school's login interface.

Total Student count collect
  • To track students and schools, including the collection of student ranks beyond grades, we need to gather the total student count for all classes and mediums.
  • This information will be collected in the school module's registration/renewal section.
  • Schools must update their total student count and fees for each class and medium.
  • The total student count will be used in the student tracking system, while the fee details will be used in the reimbursement module.
  • We need to integrate this information into the section where school fees are collected.
Report View of students
Student records can be viewed in two ways:
  1. By clicking on the student’s name displayed on the tracking collection form.
  2. By accessing a separate page that provides a year-by-year summary. On this page, you can click on a specific year to display detailed information for that period.
  3. Summary will display like a tabular format which show following information:
    1. Year
    2. Class
    3. Medium
    4. RTE Students
    5. Tracking updated
    6. Update Pending
  4. When clicking on a specific year, a view panel should display the student records, including the following details: Entry Class, Entry Year, Current Class, Medium, Student Name, Date of Birth, Parent Name, Mobile Number, Address, Class Gender, and Religion.
  5. When clicking on a specific class or medium, the same information will be displayed on the same screen, but only for the selected class or medium.
  6. When clicking on "RTE Students," "Tracking Updated," or "Update Pending" sections, the list of students will be displayed based on the selected fields.
