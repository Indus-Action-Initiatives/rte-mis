RTE MIS Product Documentation

Student Management

In the Student module, the Applicant will get all the information regarding section 12(1)(c) scheme, to get the benefit of this, an applicant can apply and participate in the admission process. There should be some specific steps that applicants need to follow that are explained here.

Student Application -Dynamic Fields

In the student management module, there should be some fields which are required to be in build or pre-filled information to display in the application form. Some fields are dynamic in nature which differ for different states, some fields are static which are once deployed on form then do not need modifications and there is a predefined logic to display the list of eligible list of schools to particular applicants.

In the Student application - dynamic fields are created which may be differ for different states. In this section below list of informations are to be incorporated-

  1. Details of location - Locations information with a proper nesting should display in the student application form. Nesting should be like District → Block → GP/Municipal Corp. → …/Ward → Habitation
  2. Applicable categories - There should be two broad categories through which an application can take admission to selected schools.
    1. DG - In disadvantaged groups some sub categories come which are SC (Scheduled Caste), ST (Scheduled Tribe), Disabled (40% or more),Primitive Tribal Group, Forest Dweller Scheduled Caste, HIV, Orphan etc.
    2. EWS - In the economically weaker section we can see those applicants will be eligible who are economically weak. In this category major BPL certificate holder families will come.

Categories in these fields may differ for different states. Currently these fields are based on existing information and for understanding the requirements.

Student application - static fields

In the static fields section, that list of information comes which are required to update once at the time of configuration, and then not required any further changes. List of such fields are listed below-

  1. Genders - Boy, Girl, Transgender
  2. Caste - ST, SC, OBC, GEN, BC, OC
  3. Religions - Hindu, Muslim, Sikh, Christian, and others
  4. Siblings - If yes then these additional information to be collected (Sibling Name, Current Class, School, aadhar card (4digit))
  5. Single girl child - Yes / No
  6. HIV - Yes / No
  7. Orphan - Yes / No

Fields from Point d to g are optional and based on States choice, these will be visible in case the State wants these particular filtrations. There should be an option to the App admin to enable or disable these fields.

In case State goes for above categories (from point d to g) there are some certain validations to be applied those are

  1. If the applicant selects the single girl child then Orphan and Sibling should be disabled to select.
  2. If the applicant selects an orphan then single girl child and Sibling should be disabled.
  3. If the applicant selects Sibling then Orphan and single girl child should be disabled.
  4. HIV should be enabled in all situations.

Other than above categories some documents are also required to identify the applicant and the information provided by the applicant.

  1. Birth proof - ANM Registered Card, Anganwadi Card, Hospital Birth Certificate, self-attested letter etc.
  2. Address Proof - Aadhaar card, Ration card, Bank statement or passbook, Voter ID card / Voter ID card, Driver license / Driving license, Gas connection bill (which is not more than 3 months old) ), Farmer Photo Passbook (KCC Card), Registered Lease/Sale/Rent Agreement (Kirayanama), MNREGA Job Card etc.
  3. Identity Proof - Aadhaar Card, Voter ID Card / Voter ID Card, Driver License / Driving License, Kishan Photo Passbook (KCC), Ration Magazine, PDS Photo, Card, Photo Bank ATM Card, PAN Card, MNREGA Job Card etc.

Student application - List of eligible school

While applicants fill out the application form, they need to select schools for getting admission over there. Applicants need to add a preference of schools to which they want to get admitted.

  1. A list of schools will be displayed based on the current location selected (Habitation), the age of the student, and the gender of the student.
  2. Firstly based on habitation selected, select and shortlist the mapped schools.
  3. Then based on the age criteria entered, again make a filter on selected schools, shortlist only those schools who have declared entry class on which applicant is eligible to get admission.
  4. Now based on the gender selected for applicants, one more level of filter is applied to selected schools, like if the selected gender is Boy then girl's schools should not display, if the selected gender is Girl then boy's schools should not display, and if selected gender is Transgender then all selected schools will display.
  5. Once the list of schools is displayed (after applying all the above filtration), there should be a column which is to select the preference and set priority to the schools to get admission.
  6. To set the preference there should be a dropdown list, which should be filled with numbers that contain the maximum number of schools displayed on the form.
  7. Also, duplication preferences should not be possible.
  8. If school runs in two mediums, in that situation school should display two times for two different mediums, and it should be parents' choice to apply for both mediums or any one.

Student Application

In the Student Management module, parents apply for their kid to get the benefit of this scheme. For these a parent should follow below process:

  1. Applicants can open the application and firstly validate the mobile number of the applicant. Once the mobile number will get validated through the OTP then changes in mobile number is not possible.
  2. While validating the mobile number, from the backend check for existing mobile numbers on the database for the current academic year.
  3. If the entered mobile number does not exist in the database then a new application form will be generated and the applicant can apply for the running academic cycle.
  4. If the entered mobile number is already exists for the current academic year, then -
    1. Applicants will get two choices: either they can choose to fill an application for another child or make modification on an existing application.
    2. If the applicant chooses the first option, then a new application form will open, with pre-filled information of parents (Parent Name, mobile number, adhar card, Caste, religion) which will be in ready only.
    3. And if the applicant chooses the second option, then the applicant is redirected to the modification page, and in this situation no validation check is required as the mobile number is already validated, application will be available for modification to applicants.
    4. If multiple applications are registered with the same mobile number then a list of all applications should be displayed with basic information, and by clicking on the application number form will be available to make modifications.
  5. Fields required in student applications should be tab delimited tabs should be like - Parent information, student information, Address information, Document Details, school selection and Declaration and submit application. Details of information to be collected of the applicant are detailed below:
    1. Parent Information-
      1. Father Name
      2. Mother Name
      3. Guardian Name
      4. Adhar Card (Father) (Last 4 digit only)
      5. Adhar Card (Mother ) (Last 4 digit only)
      6. Adhar Card (Mother ) (Last 4 digit only)
      7. Adhar Card (Guardian ) (Last 4 digit only)
      8. Father/Guardian Mobile number
      9. Caste
      10. Religion
    2. Student Information
      1. Student Name
      2. Date of Birth
      3. Gender
      4. Adhar Card (Optional) (Last 4 digit only)
      5. Is Single girl child
      6. Siblings - Yes/No
        In case of yes some additional information to be collected
        1. Sibling Name
        2. Current Class
        3. School Name
        4. aadhar card (4digit)
      7. Single girl child - Yes / No
      8. HIV - Yes / No
      9. Orphan - Yes / No
    3. Address information
      1. District
      2. Block
      3. GP/Municipal Corp.
      4. …/Ward
      5. Habitation
      6. Residential address
      7. Pincode
    4. Document Details
      1. Birth Proof (For DOB verification of applicant)
      2. Identity Proof
      3. Address Proof
      4. Category proof (For which applied)
    5. school selection
      1. List of schools from List of eligible schools
      2. Selected and shortlisted list of schools to display
    6. Declaration and submit
      1. A check box for declaration from parents
      2. After a preview of whole application application should get submitted
      3. After application submission an SMS update sent to applicants registered mobile number and print option should be displayed on the portal.
Notes:
  1. By following this validation, we can reduce the opportunities to the choice center, school admins and other persons to add their own number on the application form other than parents mobile number.
  2. Also while adding the student and parents basic information and press on the next button, then check for the duplicacy of application, if it exists then ask them to make modification on existing application.
  3. To check duplicate applications check the combination of student name, student DOB, parent name and parents Aadhar number (take last 4 digits only).
  4. DOB will check on 31st March of every year. Applicants should complete 3 yrs till 31st March to apply and less than 6.5 years. ( Configurable by the state )
  5. Log should maintain for applications status changed and display on application status view.

Student Application Print

At the end of application submission / modification there should be an option to get a printout of the application. There should be two ways to get an application printed, one is immediately just after the application submission automatically print form should display either in pop up or by providing print option. Second one is there should be a menu from where users can get print out of their field application whenever they require.

To get print out of the application applicant needs to enter the application number in the search field and click on the view button. After clicking the view button a filled application form will display and an option to take printout of the application should display. From where users can take print out of the application for their further reference.

Student Application Modification

Applicants can do the modification on the application by having the OTP verification of registered mobile numbers.

  1. If applicants want to make any modification on an existing application then they firstly need to enter their registered mobile number and through the OTP they validate the mobile number.
  2. If the mobile number is validated then the applicant can see the list of applications which are registered with the same mobile number (that may be one or more then one).
  3. Now from the application list parents can select the application id on which they want to make changes.
  4. Once the applicant clicks on the application ID, then the whole application will open in the edit format and the applicant can change all information in the application.
Notes:
  1. If applicants make changes in any of these information Location details, DOB, gender of student, then selection of school should be clear and the applicant needs to reselect the schools with the preference.
  2. In the case of the second round of application, the status of the application should not be changed if any applicants do the modifications.
  3. Allotted applications and after allotment changed status applications like admitted, did not come to school and dropout applications can not make further changes on application, they will not be considered in the current academic year.
  4. If the application is in verified status and allotment is not done, then applicants can make modifications only on the school selection tab, other tabs are blocked for them, and in this situation Status of the application remains the same.

Student Application Verification

Once the application is submitted and parents physically provide the documents including application hard copy. Then the role of block admin will start. Block admin needs to verify the application and all necessary documents and then update the status on the portal.

  1. To update application status on the portal, block admin should have a panel from where a block admin can see the list of applications which they received under their block.
  2. In this panel block admin can see the whole list of applications which they received.
  3. There should be a filtration option by school wise, preference wise (optional), class wise and medium. Block admin can perform any of above filtration or combination of above filter options.
  4. List of applications will display on the basis of filtration applied.
  5. Block admin needs to select the application id to which they want to change the status.
  6. After clicking on the application id, a detailed application form will be displayed in read only mode, and below there will be two options one is to change the status and another one is to write remarks to justify the status. Updated date will by default upload to the database.
  7. In case a student application needs an modification after the application registration/modification window has been closed. In that case the block admin can make modifications on the application on behalf of the applicant.
  8. For this OTP validation is required, and in case of making changes on Location details, DOB or Gender of student then school selection is again needed to select (this can be changeable till the verification window is open, also allotted applications can not be changeable by block admins).
  9. SMS will go on registered mobile numbers each time the application status will change.
  10. Log should also be maintained and displayed on the application status view page.

Student Application Modification (by block admin)

Block admin can do modification on the application only when the application window is closed for the parents and verification process has been started. During the verification of applications, if the block admin finds any modification on the existing application that should be modifiable, in that situation by having a consent letter from the parent and by validating the application through OTP applicant can do the modification in the application.

  1. Similar to application modification made by parents (as in point 6 from student management Module), block admin can also modify every information of the application.
  2. Notes of the same point (point 6 from student management Module) should be followed here as well.
  3. If the application is modified by the block admin then application status should directly move to the verified status and an SMS notification should be sent to parents registered mobile number.
  4. There should be a declaration option before the application submission for the block admin, that is they are doing modifications on the requirement basis and with the permission of parents or applicants.
  5. After submission immediately, the print option should also be displayed on the page.

Search and filter students

  1. There should be a view panel from where admins can view the list of whole applications and should have the option to apply necessary filters to get a more productive view of applications.
  2. There should be an option in block admin’s login and District admin’s login to search a particular application by having the option to search by application id, mobile number, name of student, name of parents etc.
  3. By application status or as per selected school there should be an option to search applicants.
  4. Fields to be displayed on the view panel are - application number, student name, Father Name, Guardian Name, Mobile number.
  5. There should be another view option through which as per school selected application view can be possible.
  6. Admins can select the school name from the list and then the list of applications those who have applied for that particular school should be displayed on a different list.
