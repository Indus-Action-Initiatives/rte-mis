State Management

In the State module, the state admin has the power to manage the roles of the other government authorities and their rights, schedule the campaign cycle, trigger different rounds of lottery, send SMS to allotted students, and perform the reimbursement process to the beneficiaries' (schools) accounts.

Individual School UDISE Code Approval

District administrators validate the new udise code which was entered in the RTE database by the block administrators. Block admins individually enter the Udise code of those schools which are not registered in the RTE database because they are newly registered schools.

  1. District admins will have access to approve the newly added Udise code. District administrators can do this by visiting the Udise portal and after verifying information from there District admin will approve the Udise code.
  2. Once the Udise code is approved then only schools will be able to create accounts to participate in the campaign.
  3. If district admin found any correction on information then either they can make modifications or they ask BEOs to make necessary changes, once the changes made by BEOs then DEOs can approve the udise code.

Campaign Creation

  1. Campion creation is the section where the duration for a particular section is defined which is going to be followed in a particular campaign cycle.
  2. It should be possible that there should be two or more different cycles that may run on a particular campaign year. For this, log for each cycle and for each year should be maintained.
  3. This is the State admin's responsibility to define the duration of a cycle to follow, and that duration should also be entered as start date and end date on the campaign creation panel of MIS portal.
  4. Below are the list of sections that shows the step by step process to follow to run a campaign and to define the duration for each:
    1. School registration (Start Date - End Date)
    2. School Renewal (Start Date - End Date)
    3. School registration / Renewal verifications (Start Date - End Date)
    4. School Mapping / Modifications (Start Date - End Date)
    5. Student Application / Modifications (Start Date - End Date)
    6. Student Application Verification (Start Date - End Date)
    7. Lottery (Start Date - End Date)
    8. Post lottery admission to school (Start Date - End Date)
    9. Student tracking (Start Date - End Date)
    10. Reimbursement claim (Start Date - End Date, Session for which claim will open)

New User Creation form for Block, State, and District

  1. User creation can be made in two different levels, one at District level where District admin can create new block admins or modify on information of existing admins information.
  2. Another is at State level, where state admin can create new district admins or modify existing information.
  3. Fields that should be there in Block admin master tables are:
    1. Block Name (Alphabetic)
    2. Authority Name (Alphabetic)
    3. Mobile Number (Number only, 10 digit min and Max both, proper number check validation)
    4. Status(Active or not)(bool)
    5. Created Date (Datetime) (Auto generated, not display at UI)
    6. Updated Date (Datetime) (Auto generated, not display at UI)
    7. Deleted Date (Datetime) (Auto generated, not display at UI)
  4. Fields that should be there in the District admin master tables are:
    1. District Name (Alphabetic)
    2. Authority Name (Alphabetic)
    3. Mobile Number (Number only, 10 digit min and Max both, proper number check validation)
    4. Status(Active or not)(bool)
    5. Created Date (Datetime) (Auto generated, not display at UI)
    6. Updated Date (Datetime) (Auto generated, not display at UI)
    7. Deleted Date (Datetime) (Auto generated, not display at UI)
  5. OTP verification should be made for mobile number verification and in case of mobile number modification.
