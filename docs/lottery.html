Lottery Management

Lottery Module is one of the major modules in RTE - MIS. Through this module, applications will get allotted to a particular school. There are some predefined algorithms that should be defined and created and followed during the allotment process. Process through which lottery will perform:

  1. At first all the applications which are in approved status are considered for the lottery process. All approved applications are taken up on a pool for the allotment process.
  2. Perform randomization on selected applications, this should have possibility from the front end that user can perform randomization more then one.
  3. A random number should be allotted to all the applications and the allotment process begins on the basis of these random numbers allotted.
  4. In the algorithm, the allotment process starts by taking up the application one by one as per new random number allotted.
  5. During this process, take up the first application from the pool and check for the school selected in first preference.
  6. Now check for the availability of seats in the first preference school, if seat available then application marked as allotted to the first preference school, if no then application moves to the pending list.
  7. Now process check for the next application from the list and perform the same operation till the all application moves to either allotment status or to the pending status.
  8. Once there is no application in the random list, then those applications who are in the pending list are moved for the allotment process for the next preference of school.
  9. Again randomization should be performed to all the applications, and start the allotment process.
  10. Now take up the first application from the random list and check for availability of school in the second or next preference. If the application does not have school in the next preference list then the application will move to the no allotment list. If the application has the school in the next process then perform the next operation.
  11. In case of availability of school, application moves to the next step and checks for the availability of seat in selected schools, if seat remains application will allot to that particular school otherwise application will move to the pending status.
  12. Now the same process will repeat till all the application moves to either allotted status or to the no allotment status. Once the pending application list remains zero.
Notes:
  1. In the second round of the admission cycle, a lottery will be performed with the approved and no allotment applications, as no allotment applications will get a chance to select new schools where seats are available.
  2. In the second round lottery process, remaining seats will be calculated by including applications in admitted, allotted status. (total RTE seat - admitted - allotted).
  3. During the lottery process, always check for the school status only active schools will participate in the allotment process.
  4. Same applies to checks for aided and minority also.
  5. Seat will be checked on the basis of class, medium selected by the applicant.
  6. During the allotment process, the process check for the applicant district and school district should be the same. (In manually mapping there is possibility for mis mapping)
  7. Wait list applications will get an option for modification only for the selected schools.
  8. Application number, Student name, Father Name, Nagariya Nikaye and Gram panchayat.

Process for Implementation:

Lottery module will be implemented separately from the RTE-MIS. There should be an option in this module that this should be used separately for any specific State. Reason behind this is that, if any state wants to use only the Lottery module from the RTE-MIS, then it should be possible for that State to take up only the lottery module and get the list of allotted applications. For this we can follow the following process:

  1. An API integration should be created for both to take up the list of student applications and list of schools on which lottery process to be performed. And another API should be created from the RTE-MIS on which list of allotted students with their respective schools to be shared with the State.
  2. Ensure that applications should be allotted to their respective schools which they have selected to be admitted. So the API from the State should collect a list of students with their respective selected schools with preference.
  3. Once the API is created and integrated to the Lottery Module then the whole list will move to a temporary table and lottery operations will perform.
  4. After the allotment process is complete, all allotted applications will be stored on a temporary table and by using an API the whole allotted list will be transferred to the States database.
  5. After the transformation of data from lottery module to State’s server all temporary tables will be deleted and no records should not be stored in the RTE-MIS database.
